# Roadmap

This roadmap focuses on making the public skill library easier to evaluate, install, and improve.

## Near Term

- Collect user showcase issues with concrete projects, screenshots, and workflow notes.
- Add more benchmark examples that compare selected skill collections against baseline Codex runs.
- Keep the public core lean by improving or removing skills that do not have a clear audience-facing use case.
- Expand documentation around collection selection and common first workflows.

## Next Releases

- Publish regular tagged releases with clear changelog entries and clean release artifacts.
- Add more verified demos that show practical frontend, accessibility, motion, and QA outcomes.
- Improve contribution guidance for new skills, skill refinements, and benchmark submissions.
- Review public skill wording for consistency, trigger clarity, and professional tone.

## Ongoing Maintenance

- Validate every skill catalog change before release.
- Keep private paths, account-specific instructions, and non-public assets out of the public core.
- Prefer focused skills and installable collections over broad, unclear additions.
- Use issues and pull requests to track real user feedback and public improvements.
