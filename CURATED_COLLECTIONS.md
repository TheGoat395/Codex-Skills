# Curated Collections

Large skill libraries should be installed intentionally. This public release defaults to `public-core`: exactly 70 skills selected for frontend, website, design-engineering, QA, motion, content, and maintainer-safety workflows.

The source of truth is `curated_collections.json`. This file explains the intent behind each public collection.

## public-core

The full public release collection: 70 skills covering premium website strategy, design direction, frontend implementation, motion, QA, content polish, and maintainer safety.

This public release intentionally excludes provider/account integrations, deployment-specific skills, local OS screenshot tooling, and broad security/ops workflows that require separate review.

## taste-and-build-gates

For open-ended website, app UI, redesign, and image-to-code work before implementation.

Focus:

- infer design mode and direction dials
- lock a build-ready spec before coding
- avoid generic AI site structure
- run browser and responsive visual polish before handoff
- preserve real content and brand anchors during redesigns

## visual-direction

For design direction, imagery, typography, color, material, layout, and reference translation.

Focus:

- concrete visual decisions
- real or generated visual assets
- editorial section systems
- premium visual references without copying them
- responsive image/crop discipline

## frontend-implementation

For React, Next.js, Tailwind, shadcn/Radix, component craft, forms, state, accessibility, and UI states.

Focus:

- reusable frontend structure
- accessible primitives
- token-driven styling
- polished loading, error, empty, disabled, focus, hover, and active states
- project-local dependencies

## qa-production-core

For validation before delivery without external account integrations.

Focus:

- accessibility audit
- performance checks
- responsive layout review
- cross-browser review
- visual regression review
- SEO technical checks

## motion-core

For animation-heavy, scroll-driven, transition-heavy, and motion-polished frontend work.

Focus:

- motion as orientation and tactility
- Motion/GSAP discipline
- page and layout transitions
- jank prevention
- reduced-motion fallback behavior

## content-conversion-core

For copy, conversion flows, proof, CTA language, content structure, and brand voice.

Focus:

- specific copy instead of placeholder text
- credible proof
- conversion-critical form and checkout polish
- information architecture
- honest CTA language

## maintainer-safety-core

For safe code-changing workflow and install-safety review.

Focus:

- checkpoint before risky edits
- undo and revert workflow
- project-local installation preference
- tool/repo/package trust checks
